EVOLUCION SISTEMAS COMPUTACIONALES

Sistema Monolitico Imagen Video
Un sistema monolítico es una aplicación web o software que está diseñado como una sola unidad indivisible. Todos los componentes (interfaz de usuario, lógica de negocio y acceso a datos) están integrados en un único programa que se ejecuta en un solo servidor. Este tipo de arquitectura es simple de desarrollar y desplegar, pero puede volverse difícil de mantener y escalar a medida que la aplicación crece.
  • Ventajas: Fácil de desarrollar y probar.
  • Desventajas: Dificultad para escalar, mantenimiento complejo y falta de flexibilidad.
Sistema NTFS Imagen Video
NTFS (New Technology File System) es un sistema de archivos utilizado por Windows para organizar y almacenar datos en discos duros. Aunque no está directamente relacionado con el desarrollo web, es relevante en entornos donde se alojan servidores web en sistemas Windows. NTFS ofrece características como permisos de archivos, compresión, cifrado y soporte para grandes volúmenes de datos. Relevancia en desarrollo web: Se usa en servidores Windows para gestionar archivos de aplicaciones web.
Cliente Servidor Imagen Video

El modelo cliente-servidor es una arquitectura en la que las tareas se dividen entre proveedores de recursos (servidores) y solicitantes (clientes). En el desarrollo web, el cliente suele ser un navegador que solicita recursos (páginas, datos, etc.) a un servidor web, que procesa la solicitud y devuelve una respuesta.

    Componentes:
  • Cliente: Navegador o aplicación que realiza solicitudes.
  • Servidor: Aplicación que procesa solicitudes y devuelve respuestas (por ejemplo, Apache, Nginx).
  • Ejemplo: Un usuario accede a una página web (cliente) y el servidor envía el HTML, CSS y JavaScript.
Sistema Multinivel Imagen Video

Un sistema multinivel (o de múltiples capas) es una arquitectura que divide una aplicación en capas lógicas, cada una con una función específica. En el desarrollo web, lo más común es el modelo de tres capas:

  • Capa de presentación: Interfaz de usuario (frontend).
  • Capa de lógica de negocio: Procesamiento de datos y reglas de negocio (backend).
  • Capa de datos: Almacenamiento y gestión de datos (bases de datos).
  • Ventajas: Mayor modularidad, escalabilidad y facilidad de mantenimiento.
  • Ejemplo: Una aplicación web con un frontend en React, un backend en Node.js y una base de datos en MySQL.

Sistema Distribuido Imagen Video

Un sistema distribuido es un conjunto de componentes (servidores, bases de datos, aplicaciones) que están ubicados en diferentes máquinas o nodos, pero que funcionan juntos como un único sistema. En el desarrollo web, esto se utiliza para mejorar la escalabilidad, la disponibilidad y el rendimiento.

    Características:
  • Los nodos están conectados por una red.
  • Pueden estar geográficamente dispersos.
  • Trabajan de manera coordinada.
  • Ejemplo: Una aplicación web que usa microservicios alojados en diferentes servidores o regiones.
Sistema en la Nube Imagen Video

Un sistema en la nube es una arquitectura que utiliza recursos y servicios proporcionados por proveedores de nube (como AWS, Azure o Google Cloud) para alojar y ejecutar aplicaciones web. Estos sistemas son altamente escalables, flexibles y suelen operar bajo un modelo de pago por uso.

    Características:
  • Escalabilidad automática.
  • Acceso a servicios gestionados (bases de datos, almacenamiento, IA, etc.).
  • Alta disponibilidad y redundancia.
  • Ejemplo: Una aplicación web alojada en AWS usando EC2 para servidores, RDS para bases de datos y S3 para almacenamiento.
Sistema con Inteligencia Artificial Imagen Video

Son sistemas avanzados capaces de aprender de los datos mediante algoritmos, identificar patrones y mejorar su desempeño con el tiempo. Pueden tomar decisiones autónomas, automatizar procesos complejos, interactuar usando lenguaje natural, imágenes o voz, y adaptarse al contexto y necesidades del usuario.

    Características:
  • Usan algoritmos de aprendizaje automático para identificar patrones y mejorar con el tiempo.
  • Pueden analizar información y responder sin intervención humana directa.
  • Desde asistentes virtuales hasta sistemas de recomendación y predicción.
  • Pueden comprender lenguaje natural, imágenes y voz, haciendo la comunicación más fluida.
  • Ajustan su comportamiento según el contexto y las necesidades del usuario.
